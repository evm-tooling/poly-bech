Get started with poly-bench in just a few steps.

## Overview

poly-bench is a multi-language benchmarking framework that lets you define benchmarks once and run them across Go, TypeScript, Rust, and more with unified output and fair comparisons:

- **Define once, run everywhere** — Write one `.bench` file; poly-bench compiles to native code in each target language.
- **Fair comparisons** — Same iterations, same data, same warmup strategy across all languages ensures apples-to-apples results.
- **Rich output** — Console tables, markdown reports, JSON for CI, and SVG charts from a single run.
- **Developer experience** — Custom DSL with LSP support, auto-calibration, and portable fixtures.

You can learn more in [Why poly-bench](/docs/introduction).

### 1. Define your bench suite

Initialise a new Polybench project by running the `init` CLI command. you can do so by copying the command below into your terminal

<CodeGroup 
  tabs={[
    {
      title: "Terminal",
      language: "bash",
      code: `poly-bench init my-benchmarks --languages go,ts,rust
`
    },
  ]}
/>

<Aside type="note">
<span>Alternatively, run <code>poly-bench init</code> with no flags — it will walk you through an interactive menu to select languages. If you don't have a given language installed, poly-bench can install it for you.</span>
</Aside>

Declare a suite with the required header tokens: `suiteType`, `runMode`, and `sameDataset`. Use `performance` for timing, `timeBased` for auto-calibration, and set `sameDataset: true` when all benchmarks share the same fixture data.

<CodeGroup 
  tabs={[
    {
      title: "benchmarks/hash.bench",
      language: "bench",
      code: `declare suite hashBench performance timeBased sameDataset: true {
    description: "String Concatenation hash benchmark"
    baseline: "go"
    warmup: 100
    targetTime: 1000ms
    count: 1

    # ... setup and benchmarks next
}`
    },
  ]}
/>

### 2. Define your params

Add fixtures (shared test data) and setup blocks for each language. Fixtures ensure all implementations receive identical input. Setup blocks define imports and helpers.

<CodeGroup 
  tabs={[
    {
      title: "benchmarks/hash.bench",
      language: "bench",
      code: `declare suite hashBench performance timeBased sameDataset: true {
    description: "String Concatination hash benchmark"
    baseline: "go"
    warmup: 100
    targetTime: 1000ms
    count: 1

     setup go { //[!code focus: 29]
      import (
            "encoding/binary"
            "strings"
        )
      
      helpers {
          func concatGo(data []byte) []byte {
            n := binary.LittleEndian.Uint32(data[0:4])
            var b strings.Builder
            b.Grow(int(n))  // pre-allocate
            for i := uint32(0); i < n; i++ {
                b.WriteByte('x')
            }
            return []byte(b.String())
        }
      }
    }

    setup ts {
        helpers {
            function concatTs(data: Uint8Array): Uint8Array {
                const n = new DataView(data.buffer, data.byteOffset, 4).getUint32(0, true)
                let s = ''
                for (let i = 0; i < n; i++) s += 'x'
                return new TextEncoder().encode(s)
            }
        }
    }
}`
    },
  ]}
/>

### 3. Add your bench tests and plot your chart

Add benchmark blocks that call your helpers across languages. Use `after` blocks to generate charts from the results.

<CodeGroup 
  tabs={[
    {
      title: "benchmarks/hash.bench",
      language: "bench",
      code: `use std::charting // [!code focus: 1]

declare suite hashBench performance timeBased sameDataset: true {
    description: "String Concatination hash benchmark"
    baseline: "go"
    warmup: 100
    targetTime: 1000ms
    count: 1

     setup go { 
        import ("encoding/binary")
        
        helpers {
          import (
                "encoding/binary"
                "strings"
            )
          
          helpers {
              func concatGo(data []byte) []byte {
                n := binary.LittleEndian.Uint32(data[0:4])
                var b strings.Builder
                b.Grow(int(n))  // pre-allocate
                for i := uint32(0); i < n; i++ {
                    b.WriteByte('x')
                }
                return []byte(b.String())
            }
      }
    }

    setup ts {
        helpers {
            function concatTs(data: Uint8Array): Uint8Array {
                const n = new DataView(data.buffer, data.byteOffset, 4).getUint32(0, true)
                let s = ''
                for (let i = 0; i < n; i++) s += 'x'
                return new TextEncoder().encode(s)
            }
        }
    }

    fixture data {  //[!code focus: 20]
        hex: "64000000"
    }

    bench stringConcatBench {
        go: concatGo(data)
        ts: concatTs(data)
    }

    after {
        charting.drawTable(
            title: "String Conctenation Performance",
            output: "str-concat-table.svg"
        )
        charting.drawSpeedupChart(
            title: "String Conctenation Speedup",
            output: "str-concat-speedup.svg"
        )
    }
     }
}`
    },
  ]}
/>

Run the benchmark:

<CodeGroup 
  tabs={[
    {
      title: "Terminal",
      language: "bash",
      code: `poly-bench run benchmarks/string-concat.bench`
    },
  ]}
/>

Charts are written to the output directory (e.g. `out/hash-table.svg`).

## CLI at a glance

A quick tour of the main commands you'll use day to day.

### plot

Generate charts from existing results without re-running benchmarks. Use `plot from-file` to apply chart directives from a `.bench` file's `after` block, or use subcommands like `bar-chart`, `line-chart`, `speedup-chart`, and `table` with a results file:

<CodeGroup 
  tabs={[
    {
      title: "Terminal",
      language: "bash",
      code: `poly-bench plot from-file benchmarks/hash.bench --results out/results.json`
    },
  ]}
/>

### compile

Verify that your benchmark scripts will build across all languages without actually running them. Useful in CI or before committing:

<CodeGroup 
  tabs={[
    {
      title: "Terminal",
      language: "bash",
      code: `poly-bench compile benchmarks/hash.bench`
    },
  ]}
/>

### install and build

- **`poly-bench install`** — Install dependencies declared in `polybench.toml` into each language's runtime environment (e.g. `go get`, `npm install`, `cargo add`).
- **`poly-bench build`** — Regenerate the `.polybench/` directory, run codegen, and compile. Use this when you've changed dependencies or want to refresh the runtime environment.

<CodeGroup 
  tabs={[
    {
      title: "Terminal",
      language: "bash",
      code: `poly-bench install
poly-bench build`
    },
  ]}
/>

### add-runtime

Add a new language runtime to an existing project. Creates the runtime config in `polybench.toml` and scaffolds `.polybench/runtime-env/<lang>/`:

<CodeGroup 
  tabs={[
    {
      title: "Terminal",
      language: "bash",
      code: `poly-bench add-runtime python
poly-bench add-runtime zig`
    },
  ]}
/>

Supported runtimes: `go`, `ts`, `rust`, `python`, `c`, `csharp`, `zig`.

### add

Add language dependencies using the runtime flag (`--go`, `--ts`, `--rust`, etc.). Each dependency is appended to the corresponding section in `polybench.toml` and installed into `.polybench/runtime-env/<lang>/`:

<CodeGroup 
  tabs={[
    {
      title: "Terminal",
      language: "bash",
      code: `poly-bench add --go github.com/example/pkg@v1.0.0
poly-bench add --ts lodash@4.17.21
poly-bench add --rs serde@1.0`
    },
  ]}
/>

### The `polybench.toml` manifest

Your project manifest lives at the root and configures defaults, languages, and dependencies. When you run `poly-bench add`, it writes entries into the `[go.dependencies]`, `[ts.dependencies]`, and similar sections. Key sections:

<CodeGroup 
  tabs={[
    {
      title: "polybench.toml",
      language: "toml",
      code: `[project]
name = "my-benchmarks"
version = "0.0.1"

[defaults]
iterations = 1000
warmup = 100
languages = ["go", "ts", "rust"]

# Per-language config and dependencies
[go]
version = "1.21"

[go.dependencies]
"github.com/example/pkg" = "latest"

[ts.dependencies]
lodash = "4.17.21"

[rust.dependencies]
serde = "1.0"
tokio = { version = "1.0", features = ["rt"] }

[output]
output_dir = "out"
auto_save_results = true`
    },
  ]}
/>

| Section | Purpose |
|---------|---------|
| `[project]` | Project name and version |
| `[defaults]` | Default iterations, warmup, and language list |
| `[go]`, `[ts]`, `[rust]`, etc. | Language-specific settings (version, module path, etc.) |
| `[go.dependencies]`, `[ts.dependencies]`, etc. | Dependencies for each runtime; `poly-bench add` updates these |
| `[output]` | Output directory, report formats, auto-save behavior |
